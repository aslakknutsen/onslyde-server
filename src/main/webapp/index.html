How it works

Declaritive Markup

Everything is controlled through HTML, allowing for easy slide creation.

Voting Options

Are given through the data-option attribute. You are only allowed 2 options per poll

Fork Your Slides

After the audience votes, your slides will be forked for the majority vote. You can have any number of slides prepared for each vote option.

Send Notes

Increase engagement and learning by sending data for each slide. This is done by wrapping content with a class=``send`` node.

Get Started

Run through the example deck

  • Start the example presentation.
  • Launch the remote control. This is what your audience will see on their mobile device or laptop.
  • Navigate through each slide until you hit a survey/poll.
  • Use the remote control to vote.
  • Notice how the content you specify is sent to each remote as you move through the slides
  • Still need more info? Read this blog post.

Download the presentation

Just save the example (File->Save Page As (make sure you select the "Webpage, HTML only" option) to your computer and start building your presentation.

Setup a private session

Right now, you're viewing the example along with anyone else who happens to be viewing it. You do not want to present with this session ID (103). When you're ready to present, create a user account and you will get a unique sessionID. Use this ID in your slide deck by opening the HTML file you just saved to your computer and replacing the number "103" with your supplied session ID.



Session Analytics

You've never seen feedback like this before. View the analytics on your presentation! You'll be able to see which slides the audience liked the best and how you performed for EACH SLIDE! (analytics dashboard is currently being built, but we can provide the raw data upon request)

Presentation Frameworks

We also have a generic HTML+CSS slide deck (somewhere) if you don't like reveal - but seriously, how could you not like revealjs? :) The plan is to integrate with all presentation frameworks.
