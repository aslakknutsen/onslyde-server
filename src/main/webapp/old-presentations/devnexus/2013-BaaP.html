Client vs. Server HTML Generation

Somewhere between 2008 and 2009, generating HTML and other resources on the backend broke down.

This was mostly due to the progressive enhancement

(enhancing pages with AJAX and other performance optimizations.)

This forced developers to create better APIs and more efficient ways of delivering data (through JSON and XML).

When everything runs on the client, however, performance can be reduced.

JavaScript heavy applications are not SEO friendly

Server side pre-rendering?

Client side generation (PROs):

  • Reduces server load
  • JSON uses less bandwidth than presentation ready HTML
  • CDN Bandwidth (jQuery, BackBone, etc...)
  • Offline capability \o/

Client side generation (CONs):

  • Can reduce performance of device/browser
  • Parsing JSON and generating HTML uses more memory
  • Hardware access === battery drain
  • SEO

The most notable client side con is security

  • Web Storage is widely supported but not secure

Dont end up with a huge mess!

(Most of the time, you’ll end up with a hybrid app)
  • Dealing with separate code bases takes special care
  • JavaScript MV* frameworks help with this
